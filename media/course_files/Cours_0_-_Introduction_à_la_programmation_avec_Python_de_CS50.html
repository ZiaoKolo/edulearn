Créer du code avec Python

  • VS Code est un type spécial d'éditeur de texte appelé compilateur. En haut, vous remarquerez un éditeur de texte. En bas, vous verrez un terminal dans lequel vous pouvez exécuter des commandes.
  • Dans le terminal, vous pouvez exécuter code hello.pypour commencer le codage.
  • Dans l'éditeur de texte ci-dessus, vous pouvez saisir print("hello, world"). Il s'agit d'un programme canonique célèbre que presque tous les codeurs écrivent au cours de leur processus d'apprentissage.
  • Dans la fenêtre du terminal, vous pouvez exécuter des commandes. Pour exécuter ce programme, vous allez devoir déplacer votre curseur vers le bas de l'écran, en cliquant dans la fenêtre du terminal. Vous pouvez maintenant saisir une deuxième commande dans la fenêtre du terminal. À côté du signe dollar, saisissez python hello.pyet appuyez sur la touche Entrée de votre clavier.
  • Rappelons que les ordinateurs ne comprennent en réalité que les zéros et les uns. Par conséquent, lorsque vous exécutez python hello.py, Python interprète le texte que vous avez créé hello.pyet le traduit en zéros et en uns que l'ordinateur peut comprendre.
  • Le résultat de l’exécution du python hello.pyprogramme est hello, world.
  • Félicitations ! Vous venez de créer votre premier programme.

Fonctions

  • Les fonctions sont des verbes ou des actions que l’ordinateur ou le langage informatique sait déjà exécuter.
  • Dans votre hello.pyprogramme, la printfonction sait comment imprimer dans la fenêtre du terminal.
  • La printfonction prend des arguments. Dans ce cas, "hello, world"ce sont les arguments que la printfonction prend.

Insectes

  • Les bugs font partie intégrante du codage. Ce sont des erreurs, des problèmes que vous devez résoudre ! Ne vous découragez pas ! Cela fait partie du processus pour devenir un excellent programmeur.
  • Imaginez que dans notre hello.pyprogramme, nous avons tapé accidentellement print("hello, world"notice et que nous avons oublié la dernière )instruction requise par le compilateur. Si je fais volontairement cette erreur, le compilateur affichera une erreur dans la fenêtre du terminal !
  • Souvent, les messages d'erreur vous informeront de vos erreurs et vous fourniront des indices sur la façon de les corriger. Cependant, il y aura de nombreuses fois où le compilateur ne sera pas de ce type.

Améliorer votre premier programme Python

  • Nous pouvons personnaliser votre premier programme Python.

  • Dans notre éditeur de texte, hello.pynous pouvons ajouter une autre fonction. inputest une fonction qui prend une invite comme argument. Nous pouvons éditer notre code pour dire

    input("What's your name? ")
print("hello, world")
  • Cette modification seule ne permettra cependant pas à votre programme de sortir ce que votre utilisateur saisit. Pour cela, nous devrons vous présenter les variables

Variables

  • Une variable est simplement un conteneur pour une valeur dans votre propre programme.

  • Dans votre programme, vous pouvez introduire votre propre variable dans votre programme en la modifiant pour la lire

    name = input("What's your name? ")
print("hello, world")

    Notez que ce =signe égal au milieu de name = input("What's your name? ")a un rôle particulier dans la programmation. Ce signe égal affecte littéralement ce qui est à droite à ce qui est à gauche. Par conséquent, la valeur renvoyée par input("What's your name? ")est affectée à name.

  • Si vous modifiez votre code comme suit, vous remarquerez une erreur

    name = input("What's your name? ")
print("hello, name")
  • Le programme reviendra hello, namedans la fenêtre du terminal, indépendamment de ce que l'utilisateur tape.

  • En éditant davantage notre code, vous pouvez taper

    name = input("What's your name? ")
print("hello,")
print(name)

  • Le résultat dans la fenêtre du terminal serait

    What's your name? David
hello
David

  • Nous nous rapprochons du résultat que nous pourrions espérer !

  • Vous pouvez en apprendre davantage dans la documentation de Python sur les types de données .

Commentaires

  • Les commentaires sont un moyen pour les programmeurs de suivre ce qu'ils font dans leurs programmes et même d'informer les autres de leurs intentions concernant un bloc de code. En bref, ce sont des notes pour vous-même et pour les autres qui verront votre code !

  • Vous pouvez ajouter des commentaires à votre programme pour voir ce que fait votre programme. Vous pouvez modifier votre code comme suit :

    # Ask the user for their name
name = input("What's your name? ")
print("hello,")
print(name)
  • Les commentaires peuvent également vous servir de liste de tâches à accomplir.

Pseudo-code

  • Le pseudo-code est un type de commentaire important qui devient un type spécial de liste de tâches à effectuer, en particulier lorsque vous ne comprenez pas comment accomplir une tâche de codage. Par exemple, dans votre code, vous pouvez modifier votre code pour dire :

    # Ask the user for their name
name = input("What's your name? ")

# Print hello
print("hello,")

# Print the name inputted
print(name)

Améliorer davantage votre premier programme Python

  • Nous pouvons encore éditer notre code comme suit :

    # Ask the user for their name
name = input("What's your name? ")

# Print hello and the inputted name
print("hello, " + name)
  • Il s'avère que certaines fonctions prennent de nombreux arguments.

  • Nous pouvons utiliser une virgule ,pour transmettre plusieurs arguments en modifiant notre code comme suit :

    # Ask the user for their name
name = input("What's your name? ")

# Print hello and the inputted name
print("hello,", name)

    La sortie dans le terminal, si nous tapions « David », nous serions hello, David. Succès.

Chaînes et paramètres

  • Une chaîne, appelée a stren Python, est une séquence de texte.

  • En revenant un peu en arrière dans notre code pour revenir à ce qui suit, il y avait un effet secondaire visuel de voir le résultat apparaître sur plusieurs lignes :

    # Ask the user for their name
name = input("What's your name? ")
print("hello,")
print(name)
  • Les fonctions prennent des arguments qui influencent leur comportement. Si nous regardons la documentation, printvous remarquerez que nous pouvons en apprendre beaucoup sur les arguments que prend la fonction print.
  • En regardant cette documentation, vous apprendrez que la fonction d'impression inclut automatiquement un morceau de code end='\n'. This \n indicates that the print function will automatically create a line break when run. The print function takes an argument called end` et que la valeur par défaut est de créer une nouvelle ligne.
  • Cependant, nous pouvons techniquement fournir un argument pour endnous-mêmes afin qu'une nouvelle ligne ne soit pas créée !

  • Nous pouvons modifier notre code comme suit :

    # Ask the user for their name
name = input("What's your name? ")
print("hello,", end="")
print(name)

    En fournissant, end=""nous écrasons la valeur par défaut de endtelle sorte qu'elle ne crée jamais de nouvelle ligne après cette première instruction print. En fournissant le nom « David », la sortie dans la fenêtre du terminal sera hello, David.

  • Les paramètres sont donc des arguments qui peuvent être pris par une fonction.

  • Vous pouvez en apprendre plus dans la documentation de Python sur print.

Un petit problème avec les guillemets

  • Remarquez à quel point l’ajout de guillemets dans votre chaîne est un défi.
  • print("hello,"friend"")ne fonctionnera pas et le compilateur renverra une erreur.
  • En général, il existe deux approches pour résoudre ce problème. Tout d'abord, vous pouvez simplement remplacer les guillemets par des guillemets simples.
  • Une autre approche, plus couramment utilisée, consiste à coder comme suitprint("hello, \"friend\"") : . Les barres obliques inverses indiquent au compilateur que le caractère suivant doit être considéré comme un guillemet dans la chaîne et éviter une erreur du compilateur.

Formatage des chaînes

  • La manière la plus élégante d’utiliser des chaînes serait probablement la suivante :

    # Ask the user for their name
name = input("What's your name? ")
print(f"hello, {name}")

    Notez le fin print(f"hello, {name}"). Il fs'agit d'un indicateur spécial pour Python qui permet de traiter cette chaîne d'une manière spéciale, différente des approches précédentes que nous avons illustrées dans ce cours. Attendez-vous à utiliser ce style de chaîne assez fréquemment dans ce cours.

En savoir plus sur les cordes

  • Vous ne devez jamais vous attendre à ce que votre utilisateur coopère comme prévu. Par conséquent, vous devrez vous assurer que les saisies de votre utilisateur sont corrigées ou vérifiées.
  • Il s'avère que la possibilité de supprimer les espaces d'une chaîne est intégrée aux chaînes.

  • En utilisant la méthode stripon nameas name = name.strip(), tous les espaces à gauche et à droite de la saisie de l'utilisateur seront supprimés. Vous pouvez modifier votre code pour qu'il soit :

    # Ask the user for their name
name = input("What's your name? ")

# Remove whitespace from the str
name = name.strip()

# Print the output
print(f"hello, {name}")

    En réexécutant ce programme, quel que soit le nombre d'espaces que vous tapez avant ou après le nom, il supprimera tous les espaces.

  • En utilisant cette titleméthode, le nom de l'utilisateur serait mis en majuscules :

    # Ask the user for their name
name = input("What's your name? ")

# Remove whitespace from the str
name = name.strip()

# Capitalize the first letter of each word
name = name.title()

# Print the output
print(f"hello, {name}")
  • À ce stade, vous en avez peut-être assez de taper pythonà plusieurs reprises dans la fenêtre du terminal. Vous pouvez utiliser la flèche vers le haut de votre clavier pour rappeler les commandes de terminal les plus récentes que vous avez effectuées.

  • Notez que vous pouvez modifier votre code pour être plus efficace :

    # Ask the user for their name
name = input("What's your name? ")

# Remove whitespace from the str and capitalize the first letter of each word
name = name.strip().title()

# Print the output
print(f"hello, {name}")

    Cela crée le même résultat que votre code précédent.

  • On pourrait même aller plus loin !

    # Ask the user for their name, remove whitespace from the str and capitalize the first letter of each word
name = input("What's your name? ").strip().title()

# Print the output
print(f"hello, {name}")
  • Vous pouvez en apprendre davantage sur les chaînes dans la documentation de Python surstr

Entiers ou int

  • En Python, un entier est appelé int.
  • Dans le monde des mathématiques, nous connaissons les opérateurs +, -, *, / et %. Ce dernier opérateur %ou opérateur modulo ne vous est peut-être pas très familier.
  • Vous n'avez pas besoin d'utiliser la fenêtre de l'éditeur de texte de votre compilateur pour exécuter du code Python. Dans votre terminal, vous pouvez exécuter pythonseul. Une fenêtre de terminal vous sera présentée >>>. Vous pouvez alors exécuter du code interactif en direct. Vous pouvez saisir 1+1, et il exécutera ce calcul. Ce mode ne sera généralement pas utilisé au cours de ce cours.
  • En ouvrant à nouveau VS Code, nous pouvons saisir du texte code calculator.pydans le terminal. Cela créera un nouveau fichier dans lequel nous créerons notre propre calculatrice.

  • Tout d’abord, nous pouvons déclarer quelques variables.

    x = 1
y = 2

z = x + y

print(z)

    Naturellement, lorsque nous exécutons, python calculator.pynous obtenons le résultat dans la fenêtre du terminal de 3. Nous pouvons rendre cela plus interactif en utilisant la inputfonction.

    x = input("What's x? ")
y = input("What's y? ")

z = x + y

print(z)
  • En exécutant ce programme, nous découvrons que la sortie est incorrecte 12. Pourquoi cela pourrait-il être le cas ?

  • Nous avons vu précédemment comment le +signe concatène deux chaînes. Étant donné que votre saisie au clavier de votre ordinateur arrive dans le compilateur sous forme de texte, elle est traitée comme une chaîne. Nous devons donc convertir cette saisie d'une chaîne en un entier. Nous pouvons procéder comme suit :

    x = input("What's x? ")
y = input("What's y? ")

z = int(x) + int(y)

print(z)

    Le résultat est maintenant correct. L'utilisation de int(x)s'appelle « casting », où une valeur est temporairement modifiée d'un type de variable (dans ce cas, une chaîne) à un autre (ici, un entier).

  • Nous pouvons encore améliorer notre programme comme suit :

    x = int(input("What's x? "))
y = int(input("What's y? "))

print(x + y)

    Cela montre que vous pouvez exécuter des fonctions sur des fonctions. La fonction interne est exécutée en premier, puis la fonction externe. La inputfonction est d'abord exécutée. Puis la intfonction.

  • Vous pouvez en apprendre davantage dans la documentation de Python int.

La lisibilité l'emporte

  • Lorsque vous décidez de votre approche pour une tâche de codage, n’oubliez pas que l’on peut raisonnablement avancer des arguments en faveur de nombreuses approches du même problème.
  • Quelle que soit l'approche que vous adoptez pour une tâche de programmation, n'oubliez pas que votre code doit être lisible. Vous devez utiliser des commentaires pour vous donner, ainsi qu'aux autres, des indices sur ce que fait votre code. De plus, vous devez créer du code de manière lisible.

Notions de base sur le flotteur

  • Une valeur à virgule flottante est un nombre réel contenant un point décimal, tel que 0.52.

  • Vous pouvez modifier votre code pour prendre en charge les flottants comme suit :

    x = float(input("What's x? "))
y = float(input("What's y? "))

print(x + y)

    Cette modification permet à votre utilisateur de saisir 1.2et 3.4de présenter un total de 4.6.

  • Imaginons cependant que vous souhaitiez arrondir le total à l'entier le plus proche. En consultant la documentation Python pour round, vous verrez que les arguments disponibles sont round(number[n, ndigits]). Ces crochets indiquent que quelque chose d'facultatif peut être spécifié par le programmeur. Par conséquent, vous pouvez faire round(n)pour arrondir un chiffre à son entier le plus proche. Alternativement, vous pouvez coder comme suit :

    # Get the user's input
x = float(input("What's x? "))
y = float(input("What's y? "))

# Create a rounded result
z = round(x + y)

# Print the result
print(z)

    La sortie sera arrondie à l'entier le plus proche.

  • Et si nous voulions formater la sortie de nombres longs ? Par exemple, au lieu de voir 1000, vous souhaiterez peut-être voir 1,000. Vous pouvez modifier votre code comme suit :

    # Get the user's input
x = float(input("What's x? "))
y = float(input("What's y? "))

# Create a rounded result
z = round(x + y)

# Print the formatted result
print(f"{z:,}")

    Bien que cela soit assez cryptique, cela print(f"{z:,}")crée un scénario dans lequel la sortie zinclura des virgules où le résultat pourrait ressembler à 1,000ou 2,500.

En savoir plus sur les flotteurs

  • Comment arrondir les valeurs à virgule flottante ? Tout d'abord, modifiez votre code comme suit :

    # Get the user's input
x = float(input("What's x? "))
y = float(input("What's y? "))

# Calculate the result
z = x / y

# Print the result
print(z)

    En entrant 2x et 3y, le résultat z est 0.6666666666, allant apparemment vers l'infini comme on pourrait s'y attendre.

  • Imaginons que nous souhaitons arrondir ce nombre à l'entier inférieur. Nous pourrions modifier notre code comme suit :

    # Get the user's input
x = float(input("What's x? "))
y = float(input("What's y? "))

# Calculate the result and round
z = round(x / y, 2)

# Print the result
print(z)

    Comme on pourrait s’y attendre, cela arrondira le résultat aux deux décimales les plus proches.

  • Nous pourrions également utiliser fstringpour formater la sortie comme suit :

    # Get the user's input
x = float(input("What's x? "))
y = float(input("What's y? "))

# Calculate the result
z = x / y

# Print the result
print(f"{z:.2f}")

    Ce fstringcode cryptique affiche le même résultat que notre stratégie d’arrondi précédente.

  • Vous pouvez en apprendre davantage dans la documentation Python de float.

Déf

  • Ne serait-il pas agréable de créer nos propres fonctions ?

  • Reprenons notre code final en hello.pyle tapant code hello.pydans la fenêtre du terminal. Votre code de départ devrait ressembler à ceci :

    # Ask the user for their name, remove whitespace from the str and capitalize the first letter of each word
name = input("What's your name? ").strip().title()

# Print the output
print(f"hello, {name}")

    Nous pouvons améliorer notre code pour créer notre propre fonction spéciale qui dit « bonjour » pour nous !

  • En effaçant tout notre code dans notre éditeur de texte, recommençons à zéro :

    name = input("What's your name? ")
hello()
print(name)

    En essayant d'exécuter ce code, votre compilateur renverra une erreur. Après tout, il n'existe aucune fonction définie pour hello.

  • Nous pouvons créer notre propre fonction appelée hellocomme suit :

    def hello():
    print("hello")


name = input("What's your name? ")
hello()
print(name)

    Notez que tout ce qui se trouve en dessous def hello()est indenté. Python est un langage indenté. Il utilise l'indentation pour comprendre ce qui fait partie de la fonction ci-dessus. Par conséquent, tout ce qui se trouve dans la hellofonction doit être indenté. Lorsqu'un élément n'est pas indenté, il le traite comme s'il n'était pas à l'intérieur de la hellofonction. En l'exécutant python hello.pydans la fenêtre du terminal, vous verrez que votre sortie n'est pas exactement comme vous le souhaiteriez.

  • Nous pouvons encore améliorer notre code :

    
# Create our own function
def hello(to):
    print("hello,", to)


# Output using our own function
name = input("What's your name? ")
hello(name)

    Ici, dans les premières lignes, vous créez votre hellofonction. Cette fois, cependant, vous dites au compilateur que cette fonction prend un seul paramètre : une variable appelée to. Par conséquent, lorsque vous appelez hello(name)l'ordinateur, elle passe nameà la hellofonction sous la forme to. C'est ainsi que nous passons des valeurs aux fonctions. Très utile ! En exécutant python hello.pydans la fenêtre du terminal, vous verrez que la sortie est beaucoup plus proche de notre idéal présenté plus tôt dans ce cours.

  • Nous pouvons modifier notre code pour ajouter une valeur par défaut à hello:

    # Create our own function
def hello(to="world"):
    print("hello,", to)


# Output using our own function
name = input("What's your name? ")
hello(name)

# Output without passing the expected arguments
hello()

    Testez votre code vous-même. Remarquez que le premier hellose comportera comme vous l'attendiez, et le second hello, auquel aucune valeur n'est transmise, affichera par défaut hello, world.

  • Il n'est pas nécessaire que notre fonction soit au début de notre programme. Nous pouvons la déplacer vers le bas, mais nous devons indiquer au compilateur que nous avons une mainfonction et une fonction séparée hello.

    def main():

    # Output using our own function
    name = input("What's your name? ")
    hello(name)

    # Output without passing the expected arguments
    hello()


# Create our own function
def hello(to="world"):
    print("hello,", to)

    Cependant, cela seul créera une sorte d'erreur. Si nous exécutons python hello.py, rien ne se passe ! La raison en est que rien dans ce code n'appelle réellement la mainfonction et ne donne vie à notre programme.

  • La très petite modification suivante appellera la mainfonction et rétablira le fonctionnement de notre programme :

    def main():

    # Output using our own function
    name = input("What's your name? ")
    hello(name)

    # Output without passing the expected arguments
    hello()


# Create our own function
def hello(to="world"):
    print("hello,", to)


main()

Valeurs de retour

  • Vous pouvez imaginer de nombreux scénarios dans lesquels vous ne souhaitez pas seulement qu'une fonction exécute une action, mais qu'elle renvoie également une valeur à la fonction principale. Par exemple, plutôt que d'imprimer simplement le calcul de x + y, vous pouvez souhaiter qu'une fonction renvoie la valeur de ce calcul à une autre partie de votre programme. Ce « renvoi » d'une valeur est appelé une returnvaleur.

  • Revenons à notre calculator.pycode en tapant code calculator.py. Effacez tout le code qui s'y trouve. Retravaillez le code comme suit :

    def main():
    x = int(input("What's x? "))
    print("x squared is", square(x))


def square(n):
    return n * n


main()

    En fait, xest passé à square. Ensuite, le calcul de x * xest renvoyé à la fonction principale.
